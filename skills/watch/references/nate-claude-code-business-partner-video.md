## Condensed takeaways
- Default AI behavior optimizes for feeling productive, not necessarily for value/revenue/reality.
- Before building, run an adversarial “roast”/council: contrarian, upside, first-principles, researcher, buyer, judge.
- Good verdict shape: green-light / reshape / kill, plus the cheapest validation test in the next 48 hours.
- “Done” and “working” are different: require verification evidence before accepting completion.
- Verification loops are artifact-specific: Playwright/screenshots/forms for UI, tests/smoke checks for code, source/evidence checks for docs, buyer/reality checks for business plans.
- Long sessions rot: prefer explicit handoff + fresh context over dragging a bloated session forward.
- Fan out independent work to subagents with clean context; parent keeps synthesis and judgment.
- Goal loops need objective done criteria, caps, verification, and preferably worker/judge separation.
